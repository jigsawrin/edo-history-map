## 京都・幕末パックの確認項目

コード、データ、ライセンス、ビルド、公開前監査をすべて通した後に、外部Chromeを
1プロセスだけ起動してPC相当と390×844モバイル相当を確認する。

- 地域selectに東京・江戸と京都が表示され、京都の初期年代が幕末になる
- 京都への切替で中心・ズーム・タイトル・説明・タグライン・凡例・注意文が切り替わる
- 京都では江戸専用コントロールと3種類の歴史表示方法が非表示かつフォーカス対象外になる
- 幕末36地点が現代地図上のCanvasにカテゴリ別の色・破線で表示される
- 地点のクリック・タップで情報カードが開き、各項目と安全な出典リンクが読める
- 空白地点で京都専用のデータなし表示になり、閉じる操作後は地図へフォーカスが戻る
- 現代・幕末切替で中心・ズーム・現在地を維持し、地域切替では現在地だけを維持する
- 高速切替、重複選択、読み込み失敗後の再試行で旧地域データや出典が混在しない
- 地図のドラッグ、ホイール、ダブルクリック、キーボード、モバイルのスワイプ・ピンチが動く
- PC・モバイルともツールバー、凡例、情報カードが画面外へ不自然にはみ出さない
- Console例外、Service Worker登録、意図しない外部通信がない

Canvas地点には個別DOMフォーカスがないため、キーボードだけで地点を直接選択する
確認項目には含めない。地域・年代・表示・現在地・ダイアログ・カードの閉じる操作と
フォーカス表示はキーボードで確認し、地点検索・一覧による代替は今後の課題とする。

物理モバイル実機やスクリーンリーダーを使用できなかった場合は、公開判断で
「未確認」と明記し、自動化・CDP相当の確認と混同しない。

## ローカル確認

1. `npm run build` と `npm run preview` を実行する。
2. ChromeまたはEdgeで表示し、江戸後期を選択する。
3. 江戸地名ポイントをクリックし、地名入りの情報カードだけが開くことを確認する。
4. ポイントのない場所をクリックし、「登録された歴史地名データがありません」が
   表示されることを確認する。
5. 地図のドラッグ、ホイールズーム、ダブルクリックズームを確認する。
6. 現代と江戸後期を往復し、中心・ズームを維持したまま、江戸後期へ戻した後も
   ポイントをクリックできることを確認する。
7. 「歴史背景＋江戸地名」「現代地図と比較」「現代地図＋江戸地名」の全モードで
   ポイントをクリックできることを確認する。
8. 比較用の現代基図不透明度と歴史地点不透明度を個別に変更する。
9. 町家領域が江戸市街地周辺に限定して表示され、ON/OFFと専用不透明度が動くこと、
   地名ポイントのクリックを妨げないことを確認する。
10. 江戸末期海岸線が東京湾岸付近に破線で表示され、内陸全体や旧江戸城中心部を
    不自然に覆わないこと、経緯度逆転や世界規模への飛びがないことを確認する。
11. 海岸線ON/OFF・不透明度と町家ON/OFF・不透明度が相互に影響せず、地名クリック、
    ドラッグ、ホイール・ダブルクリックズームを妨げないことを確認する。
12. 高速な年代・3表示モード切替後に旧レイヤーが残らず、reduced motionでは
    フェードしないこと、中心・ズーム・現在地表示を維持することを確認する。

390×844程度のモバイルviewportでも、海岸線表示・ON/OFF、ポイントのタップ、
スワイプ、ピンチズーム、ツールバーの折り返し、凡例・情報カードが画面外へ
はみ出さないことを確認する。可能なら
実機でも同じ項目を確認し、実機未確認の場合は公開判断の記録へ明記する。

## GitHub Pagesとキャッシュの切り分け

1. Pagesデプロイが対象のmainコミットで成功したことをActionsで確認する。
2. 通常URLと `?verify=<mainのコミットSHA>` を付けたURLを別々に取得する。
3. 両方のHTMLに現在の表示文言があり、参照するハッシュ付きJS/CSSが一致することを
   確認する。
4. HTMLが参照するJS/CSSと4つのGeoJSONがHTTP 200で、各GeoJSONのSHA・
   Feature数・geometry種別が台帳と一致することを確認する。
5. CSPが維持され、HTTP資源の混在や未承認の外部通信先がないことを確認する。
6. `/service-worker.js` が404で、ブラウザにもService Worker登録がないことを
   確認する。
7. 表示が古い場合は、先に `Ctrl+Shift+R`（macOSは `Cmd+Shift+R`）で強制再読み込みし、
   次にプライベートブラウズで確認する。クエリ付き取得も古ければPages/CDN側、
   クエリ付き取得だけ新しければ通常URLまたはブラウザ側のキャッシュとして記録する。

確認日時、ブラウザとバージョン、viewportまたは実機名、main SHA、Pages run URL、
確認できなかった操作を公開判断の記録へ残す。
