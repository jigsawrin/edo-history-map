# 静的地点一覧

## 目的と公開URL

静的地点一覧は、JavaScriptやCanvas地図を利用しない場合にも、承認済み地点の名称、
分類、補助情報、出典へ到達するためのHTML導線である。

- `/places/`: 地域選択、出典、プライバシー説明
- `/places/edo/`: 東京・江戸の1ページ目
- `/places/edo/page-2.html` ～ `page-88.html`: 東京・江戸の続き
- `/places/kyoto/`: 京都・幕末36件

地図本体のツールバーと`noscript`から`/places/`へリンクする。一覧から地図へ戻る
リンクも各ページに置く。

## ページ分割

東京・江戸8,788件は1ページ100件、全88ページとし、最終ページは88件である。
JavaScriptなしでもHTMLの転送量とナビゲーション量を抑えつつ、一覧性を保つための
固定値である。京都・幕末36件は1ページに収める。ページ順と地点順は既存検索の
空検索順を正規化して決定し、名称の読みを推測しない。

## 生成と再現性

```bash
npm run build:static-places
npm run audit:static-links
```

`scripts/build-static-place-pages.mjs`は`public/data/`の4 GeoJSONと、京都の表示用語を
定義する`src/kyoto-place-presentation.json`だけを読み、`dist/places/`を生成する。
生成前に4データのSHA-256、Feature数、必須属性、座標範囲、出典IDを検証する。
入力、並び順、テンプレートが同じなら出力も同じで、時刻を埋め込まない。

`dist/places/manifest.json`には入力SHA、件数、ページ数、各HTML/CSSのSHA-256を記録する。
生成物はビルド成果物でありGit管理しない。`npm run build`はViteビルド後に静的一覧を
生成し、リンク監査まで実行する。

## データと表示用語の整合

- 東京・江戸は`public/data/edo-places.geojson`の既存属性だけを表示する。
- 京都・幕末は`public/data/kyoto-bakumatsu-places.geojson`と既存の出典レジストリを使う。
- 京都の分類、位置根拠、現地状態、位置精度の表示名は、地図カード、検索結果、静的一覧で
  同じJSON定義を共有する。
- 新しい史実、読み、座標、要約、分類は生成処理で推測・追加しない。

## HTML・リンクの安全性

データ由来の文字列は制御文字を除去し、`&`、`<`、`>`、`"`、`'`をHTMLエスケープする。
外部URLはHTTPS、許可済みホストとパス、認証情報なしを検証する。外部リンクは
`target="_blank" rel="noopener noreferrer"`と「外部サイト」の可視文言を持つ。

静的ページのCSPはスクリプト、画像、フォント、外部接続を禁止する。`script`、フォーム、
画像、iframe、インラインイベント属性を生成しない。監査はさらに、内部リンクとアンカー、
重複ID、空リンク、見出し順、現在ページ、前後ページ、件数、CSS、manifestの全ハッシュを
検査する。CI、CodeQL、Pagesデプロイ、公開前監査の各経路でこの監査を実行する。

## アクセシビリティとプライバシー

各ページは日本語`lang`、固有title・description、スキップリンク、`main`、階層的見出し、
通常リンク、現在ページのテキストを持つ。CSSはキーボードフォーカス、200％拡大、
狭い画面、強制カラー、印刷を考慮する。

一覧はJavaScript、位置情報、フォーム、Cookie、ブラウザストレージ、外部画像・フォント、
アクセス解析を使わない。GitHub Pagesによる配信と、利用者が明示的に選んだ出典サイトへの
移動は通信を伴う。

## 制約

静的一覧は地点の文字情報と出典への導線であり、地図上の距離、方向、重なり、周辺環境、
Canvasレイヤーを再現しない。検索フォームも提供しない。すべてのスクリーンリーダーや
物理モバイル実機での完全な動作を保証するものではない。
