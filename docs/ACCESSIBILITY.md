# アクセシビリティ設計

## 現在の操作とフォーカス管理

- 地図は `region` として識別し、ラベルと説明で矢印キーによる移動、
  `+` / `-` キーによるズームを案内する。
- 年代、基図、歴史表示方法、比較用の現代基図不透明度、歴史地点不透明度、
  現在地、出典、プライバシーは
  地図とは独立した標準HTMLコントロールとしてキーボード操作できる。
- 年代変更はコントロールを再生成せず、操作中のフォーカスを維持する。
- クロスフェードは `prefers-reduced-motion: reduce` で無効化する。
- 江戸地名Canvasはクリック・タップを受け取る一方、背景paneと現在地paneは
  ポインターイベントを受けず、地図のドラッグ・ズーム操作を妨げない。
- 地点情報カードは `aria-live="polite"` と `aria-atomic="true"` を使い、
  表示時に地図からフォーカスを奪わない。閉じるボタンを操作した後は
  地図へフォーカスを戻す。
- JavaScriptを利用できない場合の説明と外部地図サービスの案内を維持する。

## Canvas描画の制約

歴史地点は8,788件あるため、専用paneのCanvasへ描画する。各地点の
クリック判定はLeafletのインタラクティブレイヤーとして維持するが、
Canvas上の個々のマーカーは独立したDOM要素ではなく、スクリーンリーダーや
キーボードだけで全地点を選ぶための代替操作にはならない。

## 将来の地名検索・一覧表示

地図を直接操作できない利用者へ同等の地点情報を提供するため、将来の検索と
一覧は次の要件を満たす必要がある。

- ラベル付き地名検索と、年代・分類条件に同期した結果一覧を提供する。
- 件数、読み込み状態、該当なしをスクリーンリーダーへ通知する。
- 結果を標準のリンクまたはボタンとして、キーボードだけで選択できるようにする。
- 結果選択時は地図表示と情報カードを同期し、カード内容を読み上げる。
- カードを閉じた後は、地図または検索結果の選択元へフォーカスを戻す。
- 8,788件を一度にDOMへ追加せず、ページ分割または段階表示を使う。
- 地図を表示・操作できない状態でも、地点詳細と出典へ到達可能にする。

実装時は自動テストに加え、実ブラウザでキーボード、画面拡大、主要な
スクリーンリーダーを使い、フォーカス順序と重複読み上げを確認する。
