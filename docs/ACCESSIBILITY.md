# アクセシビリティ設計

## 現在の操作とフォーカス管理

- 地図は `region` として識別し、ラベルと説明で矢印キーによる移動、
  `+` / `-` キーによるズームを案内する。
- 年代、基図、歴史表示方法、江戸末期海岸線ON/OFF・不透明度、
  町家領域ON/OFF・不透明度、比較用の現代基図不透明度、歴史地点不透明度、
  現在地、出典、プライバシーは
  地図とは独立した標準HTMLコントロールとしてキーボード操作できる。
- 地域selectはlabelと関連付ける。現在版は有効地域が2件のため表示し、地域が1件になった場合だけ非表示かつフォーカス対象外にする。
- 地域変更は専用の`aria-live`領域で通知する。年代selectを更新してもlabel関連付けを維持し、情報カードを閉じた後は地域selectへフォーカスを戻す。
- 年代変更はコントロールを再生成せず、操作中のフォーカスを維持する。
- 歴史地点データがある年代では、地域別の「地点を検索」ボタンと検索・地点一覧を表示する。
  現代年代へ切り替えるとパネルを閉じて状態を消し、パネル内にフォーカスがあった場合は
  年代selectへ戻す。
- クロスフェードは `prefers-reduced-motion: reduce` で無効化する。
- 江戸地名と京都・幕末地点のCanvasはクリック・タップを受け取る一方、背景paneと現在地paneは
  ポインターイベントを受けず、地図のドラッグ・ズーム操作を妨げない。
- 町家領域は`interactive: false`で情報カード対象にせず、文字付き凡例と破線輪郭を
  併用して色だけに依存しない。出典リンクは標準HTMLリンクとして操作できる。
- 海岸線も`interactive: false`・`pointer-events: none`の専用paneとし、寒色だけでなく
  破線と文字付き凡例で識別する。町家とは別のスライダー・paneで不透明度を管理する。
- 地点情報カードは `aria-live="polite"` と `aria-atomic="true"` を使い、
  表示時に地図からフォーカスを奪わない。閉じるボタンを操作した後は
  地図へフォーカスを戻す。
- JavaScriptを利用できない場合の説明と外部地図サービスの案内を維持する。

## Canvas描画の制約

東京・江戸の歴史地点8,788件と京都・幕末の36地点は、専用paneのCanvasへ描画する。各地点の
クリック判定はLeafletのインタラクティブレイヤーとして維持するが、
Canvas上の個々のマーカーは独立したDOM要素ではなく、スクリーンリーダーや
キーボードだけで全地点を選ぶための代替操作にはならない。

## 地域別検索・地点一覧による代替操作

地域別検索・地点一覧は、Canvas地点をキーボードおよび通常のHTML操作から選択するための
代替導線である。検索結果は通常のbuttonとして提供し、ラベル、フォーカス管理、
`aria-live`通知を自動テストとキーボード操作で確認している。

- 東京・江戸の江戸後期では江戸地名8,788件、京都の幕末では史跡36件を対象にする。
- 地点名の部分一致と実在分類selectを使え、空検索では全地点を決定的な名前順で表示する。
- 1ページ50件で分割し、DOMには最大50件だけを置く。前後ページは通常のbuttonで操作する。
- 結果は通常のHTML `button`で、地点名、分類、補助情報、京都では位置精度を読み取れる。
- EnterまたはSpaceで選ぶと現在ズームを維持して地図を移動し、地図マーカーと同じ情報カードを表示する。
- 開くと検索入力へフォーカスし、閉じるまたはパネル内のEscapeで開くボタンへ戻る。
- 読み込み、結果件数、空結果、地点選択は専用の`aria-live="polite"`領域で通知する。
- 地域変更では語、分類、ページ、選択状態を消して新地域の文言と実在カテゴリへ更新する。
- 検索語、分類、ページ、選択地点はページ内メモリだけで扱い、保存・URL追加・外部送信をしない。

## 残る制約

検索一覧はCanvas地点の代替経路を改善するが、アプリ全体が完全にアクセシブルに
なったことを意味しない。地図そのものの空間関係、Canvas上の全レイヤー、視覚的な
重なりは一覧だけでは同等に伝わらない。JavaScriptや地図を利用できない状態でも
全出典へ到達できる静的一覧は未提供である。

自動DOM・ARIAテストと外部Chromeによるキーボード操作、200％拡大、
モバイル相当表示は確認している。ただし、これらはスクリーンリーダーによる
実際の読み上げ確認とは区別する。

Windows Narratorによる実際の読み上げ確認は、自動操作環境の安全機構が対象URLを
確実に判定できなかったため実施していない。安全機構の無効化・回避や同じ自動操作の
反復は行っていない。Narrator確認済み、またはすべての支援技術へ完全対応済みとは
扱わない。NVDA、VoiceOver、TalkBackによる実確認も未実施である。
