# 歴史テーマ索引

## 目的と範囲

歴史テーマ索引は、京都・幕末と滋賀・戦国の確認済み地点を、人物、事件・戦い、
勢力・組織、概念的な歴史テーマからたどる補助導線である。テーマは人物伝や因果関係を
網羅するデータベースではなく、既存地点の説明と地点別出典で直接確認できる関係だけを扱う。

東京・江戸8,788地名は人物・事件との関係を表す構造化フィールドを持たないため、
推測で関連付けず対象外とした。京都と滋賀は時代が異なるため、同名人物・政権・勢力を
時代横断で同一視しない。地域横断5テーマは「政治拠点」「寺社と権力」など、
地点の性質を比較できる概念テーマに限定する。

## 固定データと検証

`data-curation/historical-themes.json`が唯一のテーマ定義である。21テーマ、87関係を収録し、
各関係は既存のdataset ID、place ID、関係種別、独自の短い説明、地点に既に属するsource IDを持つ。
ブラウザ実行時に追加fetchを行わない。

`src/historical-theme-registry.ts`と静的生成器は次を拒否する。

- 15～25件の範囲外、ID重複、危険キー、HTML・Markdownリンク・制御文字
- 未知のテーマ種別、関係種別、dataset ID、place ID、source ID
- 2地点未満のテーマ、同一地点の重複、地点に属さない出典
- 5件未満の地域横断テーマ

テーマと関係はIDで決定的に整列する。採否の記録は
`audit/historical-theme-review.md`に残す。

## 地図版

ツールバーの「歴史テーマ」は通常のHTMLボタンでパネルを開く。検索は地点検索と同じ
Unicode正規化を再利用し、タイトル、日本語別名、英字名、要約を対象にする。種別絞り込み、
Escapeによる閉じる操作、フォーカス復帰、aria-live通知を備える。

関連地点を選ぶと、その地点が属する地域と年代へ切り替え、既存の地点選択処理と情報カードを
再利用する。非同期読込中に別の地域・年代・テーマへ移った場合は世代番号で古い選択を無効化する。
検索語、テーマ、選択地点はURL、Cookie、localStorage、sessionStorage、ログへ保存しない。

## 静的索引

`npm run build:static-themes`は`/themes/`以下にトップ、4種別一覧、21個別ページ、専用CSSを生成する。
個別ページは既存地点アンカーと地図版へリンクし、京都・滋賀の静的地点ページには最大5件の
逆リンクを追加する。静的索引はJavaScript、フォーム、画像、外部フォント、Cookie、storageを
使わず、CSPでscript、connect、image、fontを禁止する。

`dist/places/manifest.json` schema 2はテーマ数、種別件数、地域横断数、関係数、入力SHA、
全テーマHTML/CSS SHA、逆リンク追加後の地点ページSHAを記録する。`npm run audit:static-links`は
地点91ページとテーマ26ページをまとめて監査する。

## 更新手順

1. 既存地点の説明とその地点に登録済みの出典で関係を確認する。
2. 採用・見送り理由をレビュー記録へ追記する。
3. 固定JSONだけを編集し、既存GeoJSON、地点ID、地点説明は変更しない。
4. `npm run typecheck && npm test && npm run build && npm run audit:prepublish`を実行する。
5. 外部Chromeで地図版、キーボード、モバイル相当、静的索引、JavaScript無効を確認する。

完全な人物・事件DB、系譜、因果関係、全史跡網羅、スクリーンリーダー完全対応を保証しない。

## 歴史年表との相互リンク

個別テーマページは年表の`relatedThemeIds`を逆引きし、「関連する歴史年表」を表示する。
テーマ名と年表タイトルの文字列一致では関連を生成しない。地図版でテーマ・年表から地点を
選択する処理は`navigateToCuratedHistoricalPlace`へ共通化し、固定dataset ID、地域・年代対応、
世代番号、検索モデルキャッシュ、既存情報カード、エラー通知、フォーカス復帰を共有する。
年表の対象・日付方針は`docs/HISTORICAL_TIMELINE.md`を参照する。
