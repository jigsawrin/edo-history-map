# 脅威モデル

対象: いま・むかし地図(静的 Web アプリ、GitHub Pages 配信)
作成日: 2026-07-15

## 1. システム概要と信頼境界

```mermaid
flowchart LR
  subgraph Browser["利用者のブラウザ"]
    App["アプリ(HTML/JS/CSS)"]
    Geo["位置情報API(許可制)"]
  end
  Pages["GitHub Pages(静的配信)"] -->|HTML/JS/CSS/GeoJSON| App
  App -->|タイル要求(IP+タイル座標)| GSI["地理院タイル cyberjapandata.gsi.go.jp"]
  Geo -.->|座標(メモリ内のみ)| App
```

信頼境界:

- 境界A: ブラウザ ↔ GitHub Pages(配信物の完全性は GitHub に依存)
- 境界B: ブラウザ ↔ 地理院タイル(画像のみ。CSP img-src で限定)
- 境界C: アプリ ↔ 同梱 GeoJSON(ビルド時に生成するが、実行時にも
  信頼せず検証する)
- 境界D: アプリ ↔ URL パラメータ(利用者以外が細工したリンクを
  踏む可能性があるため信頼しない)

## 2. 資産

| 資産 | 重要度 | 説明 |
|---|---|---|
| 利用者の現在地 | 高 | 最も保護すべき情報。保存・送信しない |
| 配信物の完全性 | 高 | 改ざんされると利用者へ攻撃が波及 |
| リポジトリ/CI の認証 | 高 | 漏えいすると配信物改ざんに直結 |
| データの出典・ライセンス遵守 | 高 | 法的リスク・提供元への迷惑 |
| アプリの可用性 | 中 | 静的サイトのため単純 |

## 3. 脅威と対策(STRIDE ベース)

### S: なりすまし

- 脅威: 偽サイト・偽リポジトリ。
- 対策: 正規 URL を README に明記。HTTPS 強制。カスタムドメイン不使用
  (サブドメイン乗っ取りの余地を作らない)。

### T: 改ざん

- 脅威: 依存パッケージのサプライチェーン攻撃(悪意ある更新、
  タイポスクワッティング、悪意ある postinstall)。
- 対策: 依存最小(実行時依存は leaflet のみ)、`package-lock.json` 固定、
  `npm ci`、公式レジストリのみ、`--ignore-scripts` での導入確認、
  npm audit / Dependabot / CodeQL、メジャー更新の自動適用禁止。
- 脅威: GitHub Actions 経由の改ざん(悪意あるアクション、権限の広い
  トークン)。
- 対策: アクションをコミット SHA で固定、`permissions` を最小化、
  Pages 書き込みはデプロイ専用ワークフローのみ。
- 脅威: GeoJSON データの差し替え・破損。
- 対策: 実行時にスキーマ・件数・サイズ・座標範囲を検証し、
  不正なら歴史レイヤーを表示しない(地図閲覧は継続)。
- 脅威: 権利未確認または改ざんされた古地図画像の公開物への混入。
- 対策: 歴史画像の実行時レジストリは初期状態で空とし、DATA_SOURCES.yml の
  approved判定、再配布・改変・切り抜き・位置合わせ・タイル化の許可、出典、
  年代、範囲、ローカルパス、SHA-256を公開前監査で検査する。

### R: 否認 — 該当が小さい(ユーザー操作を記録しない設計のため)

### I: 情報漏えい

- 脅威: 現在地の漏えい(保存、URL、ログ、解析)。
- 対策: getCurrentPosition を明示操作時に1回のみ。メモリ内のみで使用。
  ストレージ/URL/コンソールへ書き込むコードを持たない(テストで検証)。
  解析・トラッカー・外部エラー送信を導入しない。
- 脅威: リポジトリへの秘密情報・個人情報・絶対パスの混入。
- 対策: 公開前監査スクリプト(scripts/prepublish-audit.mjs)で
  Git 追跡ファイル・履歴・ビルド成果物を検査。secret scanning +
  push protection を有効化。ソースマップは本番ビルドに含めない。
- 脅威: エラーメッセージによる内部情報の露出。
- 対策: ユーザー向けエラーは固定文言のみ(パス・URL・スタックを含めない)。

### D: サービス妨害

- 脅威: 巨大データ・不正データによるブラウザのハング。
- 対策: サイズ上限(8MB)・件数上限(10000)・文字列長上限(300)。
  Content-Length の事前検査。fetch タイムアウト(15秒)。

### E: 権限昇格 / コード実行(XSS 等)

- 脅威: データ由来文字列(地名等)による XSS。
- 対策: `textContent` のみで挿入(`innerHTML` は ESLint で禁止)。
  CSP `script-src 'self'`。インラインスクリプトなし。
- 脅威: テーマ定義からの未知ID、出典差し替え、プロトタイプ汚染、静的HTML注入。
- 対策: 固定JSONを通常オブジェクト・許可enum・既存地点ID・地点所属source IDで厳格検証し、
  危険キー、HTML、Markdownリンク、制御文字を拒否する。静的出典URLはHTTPSかつ固定origin許可リスト、
  全文字列をHTMLエスケープし、テーマ静的CSPはscript/connect/image/fontを禁止する。
- 脅威: 表示文字列からの誤った日付推測、不明月日の補完、旧暦の無根拠変換、年表からの任意URL注入。
- 対策: 年表日付は独立JSONへ明示登録し、`Date`や正規表現で生成しない。precisionとcalendarBasis、
  月日範囲、order、既存地点・テーマ・地点所属source IDを検証する。URLフィールドを許可せず、
  静的リンクは固定内部経路と既存出典レジストリだけから生成する。
- 脅威: `javascript:` / `data:` URL、危険な外部リンク。
- 対策: リンクは https + 許可リストドメインのみ。`noopener noreferrer`。
- 脅威: URL パラメータ経由の注入。
- 対策: 許可リスト方式(キーも値も固定リスト照合。入力値そのものを
  DOM・URL に使わない)。
- 脅威: プロンプトインジェクション(AI 機能悪用)。
- 対策: 本アプリは LLM 機能を持たず、利用者入力を LLM へ送る経路が
  存在しない。開発時も外部コンテンツ内の命令には従わない運用とする
  (CONTRIBUTING.md に明記)。

## 4. 受容するリスク(残存リスク)

- GitHub Pages / 地理院タイル基盤の侵害(提供元の管理領域)。
- 利用者のブラウザ・端末・拡張機能の侵害。
- CSP を meta タグで配信する制約(HTTP ヘッダーは GitHub Pages では
  設定不可。frame-ancestors 等ヘッダー限定のディレクティブは
  効かないため、X-Frame-Options 相当の保護は限定的)。
- タイル要求から表示地域(現在地表示中はその周辺)がタイル提供者に
  伝わること(地図サービスの性質上不可避。プライバシー文書に明示)。

## 古地図ラスターパックの脅威と対策

候補台帳は実行時URLレジストリではなく、調査証跡である。HTTPSの個別資料URLだけを許可し、candidateId・資料URLの重複、10件未満、3機関未満、approvedの商用・再配布・加工・切り抜き・位置合わせ・タイル化フラグ欠落を監査で拒否する。候補approvedだけではpublic配信を許可せず、既存のsource承認、raster定義、manifest、基準点、誤差評価の二重ゲートを維持する。

- 任意URL、query/hash由来raster ID、外部画像originを使わず、固定レジストリの同一originパスだけを解決する。
- sourceとrasterのapproved二重ゲート、region/era/attribution一致を必須にし、pending/rejectedを拒否する。
- manifestはXYZ、PNG/lossless WebP、256pxに限定し、絶対パス、`..`、colon、backslash、重複、欠損、orphan、symlink、SVG、HTML、ZIP、JPEG、不正magic bytes・寸法・SHA・bytesを拒否する。
- 個別5MiB、総100MiB、2万タイル、zoom 22を上限とし、ZIPを扱わないことで展開爆弾経路を持たない。
- 原本と一時加工物はGit除外し、approvedタイルと小さな監査metadataだけを公開候補にする。
- manifestはメモリ内Promiseで再利用し、地域tokenと古地図世代番号で古い非同期完了を破棄する。Service Worker、storage、blob URLは使わない。
- 原本の不適切表現は画像改変や自動置換をせず、文脈注意を別テキストで表示する。
